lamina.time documentation

advance
(advance _)
Advances to the next task. Returns false if there are no remaining tasks.

advance-until
(advance-until _ timestamp)
Advances across all tasks that occur before or on the given timestamp.

days
(days n)
Converts days -> milliseconds

format-duration
(format-duration n)
Returns a formatted string describing an interval, i.e. '5d 3h 1m'

hours
(hours n)
Converts hours -> milliseconds

hz
(hz n)
Converts frequency -> period in milliseconds

invoke-at
(invoke-at timestamp f)(invoke-at task-queue timestamp f)
Delays invocation of a function until 'timestamp'.

invoke-in
(invoke-in delay f)(invoke-in task-queue delay f)
Delays invocation of a function by 'delay' milliseconds.

invoke-repeatedly
(invoke-repeatedly period f)(invoke-repeatedly task-queue period f)
Repeatedly invokes a function every 'period' milliseconds, but ensures that the function cannot
overlap its own invocation if it takes more than the period to complete.

The function will be given a single parameter, which is a callback that can be invoked to cancel
future invocations.

milliseconds

minutes

nanoseconds
(nanoseconds n)

non-realtime-task-queue
(non-realtime-task-queue)(non-realtime-task-queue start-time discard-past-events?)
A task queue which can be used to schedule timed or periodic tasks at something
other than realtime.
now
(now)(now clock)
Returns the current time, in milliseconds since the epoch.

period
(period)
Returns the current default period for recurring actions.

seconds
(seconds n)
Converts seconds -> milliseconds

task-queue
(task-queue)
Returns the current task queue. Defaults to a real-time task queue.

with-period
(with-period period & body)
Sets the default period within the inner context.

with-task-queue
(with-task-queue q & body)
Executes the body within a context where 'q' is the task-queue.
